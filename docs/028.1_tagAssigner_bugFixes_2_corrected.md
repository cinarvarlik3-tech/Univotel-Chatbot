## 0. Why this corrected spec exists

Spec 028's root-cause analysis is correct and its §3.1 "new function + Router-computed
mention" direction is correct. **But its §3.2 minimal call-site swap, applied as written,
does not fix any of the 4 target cases.** This was verified end-to-end against the live DB
on 2026-07-16 (project `zowqgitrhpdyowikgcdm`), running the *actual* `canonicalize()` and
`resolve_university_override()` for all 4 leads:

```
case              | canonicalize(lead words) | resolve_university_override() | spec expects
cw641  Bahçeşehir | parent_only              | Bahçeşehir Üni. - Kuzey        | bilinmiyor-kampus  ← FAIL
cw237  Topkapı    | parent_only              | Topkapı Üni. - Topkapı         | bilinmiyor-kampus  ← FAIL
cw1072 Medeniyet  | parent_only              | Medeniyet Üni. - Ünalan/Göztepe| bilinmiyor-kampus  ← FAIL
cw1049 Boğaziçi   | parent_only              | Boğaziçi - Ana Kampüs          | bilinmiyor-kampus  ← FAIL
cw909  Yeditepe   | campus -> Yeditepe       | Yeditepe Üniversitesi          | resolves ✓ (unchanged)
cw1217 sbü        | campus -> Sağlık Bil.    | Sağlık Bilimleri Üniversitesi  | resolves ✓ (unchanged)
```

`canonicalize()` on the lead's own words correctly returns `parent_only` for all 4 — the
deterministic scan works exactly as 028 intends. The failure is one layer up:
`resolve_university_override()` **discards** that `parent_only` verdict and returns the
hallucinated belt.

### Root cause of the gap

`resolve_university_override`'s `PARENT_ONLY` branch
(`app/tagassigner/university_canonicalizer.py:202-207`) currently reads:

```python
if canon_result.confidence == CanonConfidence.PARENT_ONLY:
    belt_resolves = (
        proposed_uni.strip() not in _UNI_SENTINELS
        and resolve_university_list_value(proposed_uni, label_map).university_id is not None
    )
    return proposed_uni if belt_resolves else UNIVERSITY_CAMPUS_AMBIGUOUS
```

This "don't let an ambiguous mention downgrade a resolvable belt" rule was **correct in
spec 027**, when `mention` was the LLM's own flaky `university_mention` echo — you did not
want a stale echo to nuke a good belt guess.

But 028 changes what `mention` *is*: it becomes the **deterministic scan of the lead's own
inbound words**, which is authoritative. And confirmed against the live label map, each of
the 4 hallucinated belt strings is an **exact `university_chatwoot_label_map` row**:

| LLM belt | exact label-map row? | parent campuses |
|---|---|---|
| `Bahçeşehir Üniversitesi - Kuzey` | ✅ | 3 |
| `Topkapı Üniversitesi - Topkapı` | ✅ | 3 |
| `Medeniyet Üniversitesi - Ünalan/Göztepe` | ✅ | 3 |
| `Boğaziçi - Ana Kampüs` | ✅ | 2 |

So `belt_resolves` is **always `True`** for a well-formed hallucinated campus → the branch
returns the hallucination → `bilinmiyor-kampus` is never emitted. The safety net is still
checking the LLM's answer against a rule designed to protect the LLM's answer.

**The fix therefore requires a logic change inside `resolve_university_override`, not only
the §3.2 call-site swap.** When the mention is the trustworthy deterministic scan, a
`PARENT_ONLY` verdict must force `bilinmiyor-kampus` *unconditionally* — the belt must not
be able to rescue it.

## 1. Governing principles (unchanged from 027/028)

- **Model-agnostic.** Identical across `gemini-2.5-flash`, `gpt-5.4-mini`, `haiku-4.5`.
- **No reliance on higher compute.** Do not raise `LLM_REASONING_EFFORT`.
- **Scale to 10–20k conversations/sweep.** No new per-conversation unbounded DB/API calls.
- **Never invent a campus.** For a multi-campus institution named without a campus,
  `bilinmiyor-kampus` is the only correct output (product decision, 2026-07-15).

## 2. The fix — three coordinated changes

### 2.1 New function: `extract_university_phrase_from_messages` (unchanged from 028 §3.1)

In `app/tagassigner/university_canonicalizer.py`:

```python
def extract_university_phrase_from_messages(messages: list[Message]) -> Optional[str]: ...
```

- Pure function (no I/O; takes already-loaded `messages`).
- **Inbound messages only** — `msg.message_type == "inbound"` (mirror the scan in
  `app/tagassigner/fiyat_soruyor.py:73-80`). This is a **hard requirement**, not an
  optimization: it is the mechanism preventing false positives on university names inside
  bot pitch templates (e.g. cw1072's outbound "…Yeditepe, Acıbadem vb…"). Do **not** scan
  outbound/bot text.
- Concatenate/join inbound message `content` (in chronological order) into one phrase so a
  lead who splits "Topkapı üniversitesi" and "Altunizade kampüsü" across two inbound
  messages (cw430) is still seen as one phrase. Return `None` when there is no inbound text.
- Hand the concatenated inbound text to the existing `canonicalize()` — do **not** build any
  new keyword matcher here; `canonicalize()` already runs `scan_entities_by_ngram` +
  `token_containment` internally. This function's only job is to produce the right *input*
  string. (This function does not itself call `canonicalize`; it returns the phrase and the
  Router passes it on — see 2.3. Keeping it phrase-only keeps it trivially unit-testable.)

> **Implementation caution (import cycle):** `university_canonicalizer.py` must import
> `Message` from `app.db.models`. `fiyat_soruyor.py` already does this without issue, so the
> pattern is safe — but confirm no new circular import is introduced when you add it.

### 2.2 Logic change: make a deterministic mention authoritative in `resolve_university_override`

This is the change spec 028 missed. Add a `mention_is_authoritative` flag; when set and the
canonicalization is `PARENT_ONLY`, force `bilinmiyor-kampus` and skip the belt-preservation.

**New signature:**

```python
def resolve_university_override(
    proposed_uni: str,
    mention: Optional[str],
    label_map: list[tuple[uuid.UUID, str]],
    universe: UniversityUniverse,
    mention_is_authoritative: bool = False,   # NEW — default preserves 027 behavior
) -> str:
```

**New body (only the PARENT_ONLY branch changes; everything else identical):**

```python
canon_mention = mention if mention and mention.strip() not in _UNI_SENTINELS else proposed_uni

# Authoritative only when we are actually canonicalizing the caller-supplied
# (deterministic) mention — NOT when we've fallen back to the belt text because
# the mention was a sentinel/absent. Defensive: keeps the flag honest even if a
# caller passes authoritative=True with an empty mention.
used_authoritative = (
    mention_is_authoritative
    and bool(mention)
    and mention.strip() not in _UNI_SENTINELS
)

if canon_mention.strip() in _UNI_SENTINELS:
    return proposed_uni

canon_result = canonicalize(canon_mention, universe)

if canon_result.confidence == CanonConfidence.CAMPUS:
    canon_list_value = next(
        (lv for uid, lv in label_map if uid == canon_result.university_id), None
    )
    return canon_list_value if canon_list_value is not None else proposed_uni

if canon_result.confidence == CanonConfidence.PARENT_ONLY:
    if used_authoritative:
        # The lead themselves named a multi-campus institution with no resolvable
        # campus. Withhold unconditionally — the LLM belt must NOT rescue it, even
        # when the belt is a well-formed (hallucinated) campus list value.
        return UNIVERSITY_CAMPUS_AMBIGUOUS
    # LLM-echo fallback path (027 behavior, unchanged): a flaky echo must not
    # downgrade an independently-resolvable belt.
    belt_resolves = (
        proposed_uni.strip() not in _UNI_SENTINELS
        and resolve_university_list_value(proposed_uni, label_map).university_id is not None
    )
    return proposed_uni if belt_resolves else UNIVERSITY_CAMPUS_AMBIGUOUS

return proposed_uni
```

**Why this is safe and sufficient:**

- **CAMPUS path unchanged** — a confident deterministic campus already wins and translates
  via `label_map`. Positive controls cw909/cw1217 stay green (they are CAMPUS, not
  PARENT_ONLY).
- **PARENT_ONLY + authoritative → withhold** — fixes all 4 target cases. Verified: their
  lead words canonicalize to `parent_only`, so this branch fires and returns
  `bilinmiyor-kampus`.
- **PARENT_ONLY + not-authoritative → 027 belt-preservation** — retained for the
  LLM-echo/other-provider fallback path, so `test_override_parent_only_does_not_downgrade_
  confident_belt` still passes unchanged.
- **NONE → belt as-is** — when the lead's own words yield no signal at all, deferring to the
  LLM belt is the intended 028 fallback ("only fall back when the deterministic scan finds
  nothing"). This does **not** reopen the original bug: the original bug required the belt to
  self-canonicalize to CAMPUS; here, for a named multi-campus school the deterministic scan
  returns PARENT_ONLY (not NONE), so the authoritative branch fires first.

**Intentional behavioral consequence to accept (call out to product if unsure):** an
authoritative `PARENT_ONLY` now overrides *even a belt that named a real campus*, whenever
the deterministic matcher could not itself confirm that campus (same class as cw430
Altunizade — a campus not present in the DB). Per the 2026-07-15 policy ("never invent a
campus; `bilinmiyor-kampus` is not a failure state"), withholding is correct here. This is a
deliberate trade of "occasionally withhold a campus the LLM happened to guess right" for
"never write a hallucinated campus." Verify no *positive* campus case regresses (§4).

### 2.3 Call-site change in `router.py` (`apply_tagassigner_result`, ~L208-211)

Replace:

```python
universe = await get_university_universe()
override_uni = resolve_university_override(
    proposed_uni, result.university_mention, label_map, universe
)
```

with:

```python
universe = await get_university_universe()
messages_for_uni = full_history_messages
if messages_for_uni is None:
    messages_for_uni = await queries.get_messages_for_conversation(conversation_id)
deterministic_mention = extract_university_phrase_from_messages(messages_for_uni)

override_uni = resolve_university_override(
    proposed_uni,
    deterministic_mention or result.university_mention,
    label_map,
    universe,
    mention_is_authoritative=bool(deterministic_mention),
)
```

Notes:
- **Reuse the same message load — do not add a second round-trip.** The full-history fetch
  is needed a few lines below anyway for `compute_fiyat_soruyor`
  (`router.py:298-301`). Move/share that single `full_history`/`get_messages_for_conversation`
  load so both Mode B and Mode C consume one list. (Cleanest: load `full_history` once near
  the top of the attribute section and pass it to both.) The batch path arrives with
  `full_history_messages=None`, so the `is None` fallback load is required — mirror exactly
  what Mode B already does.
- `mention_is_authoritative=bool(deterministic_mention)` is the switch: `True` when we're
  passing the lead's own words, `False` when we fell through to the LLM echo (rare;
  other-provider only).
- Keep the existing `if override_uni != proposed_uni:` override-logging block.

### 2.4 Keep (do NOT remove)

- `result.university_mention` in `TagResult` / `parse_tag_result`
  (`payload_builder.py:171-174`, `router.py:379-382`) — still the fallback and a
  sweep-debugging signal.
- The optional `university_mention` prompt field (`tagassigner_prompt.md:65-71`).
- `canonicalize()`'s internal precedence tiers — proven correct; only the *input* changes.

### 2.5 Do NOT

- Do not scan outbound/bot messages in `extract_university_phrase_from_messages`.
- Do not add a per-university "default campus".
- Do not raise `LLM_REASONING_EFFORT`.
- Do not change `canonicalize()`'s tiers.
- Do not delete the non-authoritative belt-preservation branch (it still guards the

## Appendix B — Known accepted gap + ready-to-apply follow-up (decided 2026-07-16, do not implement now)

**The gap:** §2.2's authoritative `PARENT_ONLY → bilinmiyor-kampus` rule can, in principle,
discard a *correct* belt campus. This requires all three of: (1) the lead named a
multi-campus institution *and* a real campus, (2) the deterministic `match_campus` failed to
recognize that campus (matcher/spelling gap), but (3) the LLM's belt guessed that same real,
DB-valid campus correctly. This combination **did not occur in the 2026-07-15 sweep** (all 4
target cases are pure hallucinations with zero campus grounding in the lead's words) and per
the 2026-07-15 product policy ("never invent a campus; `bilinmiyor-kampus` is not a failure
state") withholding here is already the policy-preferred outcome, not a regression.

**Decision (2026-07-16):** ship §2.2 as the simple unconditional-withhold rule. Do **not**
build the grounding check below now. Revisit only if a real Case-C-shaped example shows up
in a future sweep.

**Ready-to-apply design, if/when needed — "grounded discriminator" check:**

Instead of unconditionally forcing `bilinmiyor-kampus` on authoritative `PARENT_ONLY`, keep
the belt when the lead's own words contain a **campus discriminator**: a token from the
belt-campus's `campus_label` that is *not* already part of the parent institution's own name
(subtracting institution-name tokens, not sibling-campus tokens — see below for why).

```
grounded(lead_tokens, belt_campus_label, institution_name_tokens):
    discriminator = tokens(belt_campus_label) - institution_name_tokens
    return discriminator != {} and discriminator ⊆ lead_tokens
```

Verified against live campus_label data (`university_parent_map` join `parent_universities`,
queried 2026-07-16):

| case | belt campus_label | discriminator (label − institution name) | in lead's words? | grounded? |
|---|---|---|---|---|
| cw641 Bahçeşehir | `Kuzey` | `{kuzey}` | no | **no → withhold** ✓ |
| cw237 Topkapı | `Topkapı` | `{}` (label collapses into institution name — "İstanbul **Topkapı** Üniversitesi") | n/a | **no → withhold** ✓ |
| cw1072 Medeniyet | `Göztepe` | `{goztepe}` | no | **no → withhold** ✓ |
| cw1049 Boğaziçi | `Ana Kampüs` | `{ana}` | no | **no → withhold** ✓ |
| hypothetical Case C | `Anadolu Hisarı` | `{anadolu, hisari}` | **yes**, if lead wrote "Boğaziçi Anadolu Hisarı" | **yes → keep belt** ✓ |

**Why the subtraction must be against the institution name, not sibling campus labels:** the
Topkapı case is the trap — its campus is literally named `Topkapı`, same word as the
institution ("İstanbul Topkapı Üniversitesi"). A discriminator computed against sibling
campuses only (e.g. `{Topkapı} − {Kazlıçeşme, Levent TSYD}`) would wrongly keep `{topkapi}` as
a valid discriminator and rescue the hallucination. Subtracting the *institution name's own
tokens* correctly collapses it to `{}` first.

**Cost to build (why this is deferred, not just laziness):**
- `CanonResult`/`canonicalize()` must expose `parent_university_id` on `PARENT_ONLY` (today
  it returns `None` there).
- `UniversityUniverse` cache must also hold `parent_universities.name` per parent id (one
  more static table, loaded once per sweep — consistent with existing cache design, but new
  surface).
- A `university_id -> parent_id` reverse lookup (derivable from the existing
  `campuses_by_parent` map).
- A token-set helper reusing `normalize` + `STRUCTURAL_STOPLIST`.
- New unit tests, with the Topkapı-collapse case as the mandatory anti-regression (any
  future implementer must include it — it is what makes the naive version of this check
  wrong).

*End of corrected spec.*
